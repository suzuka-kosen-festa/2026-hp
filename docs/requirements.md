## 4. 共通要素

- **背景**: コラージュ（テープや紙が切り貼りされた質感）。実装は design-system.md 参照
- **ヘッダー**:
  - PC（約900px以上）: 左にロゴテキスト、右に7項目横並びnav。hover/現在ページに赤テープ下線（Figmaデザイン準拠）
  - SP: 細い固定バー（左: 小ロゴ、右: ハンバーガー=テープ3枚）→ タップで全画面コラージュメニュー
- **フッター**: home 最下部と同一内容を全ページ共通化
- （検討中）SP画面下固定のクイックバー（map / timetable / booth）— 当日利用のショートカット

## 5. OPムービー（スプラッシュ型）

- 初回訪問時のみ全画面再生。`sessionStorage` で2回目以降スキップ
- `autoplay muted playsinline`（スマホ自動再生の必須条件）
- **スキップボタンを常時表示**（再生開始直後から）
- 再生失敗・reduced-motion・低速回線時はロゴ静止画ヒーローにフォールバック
- 素材: OP.mp4（9.8秒 / 1900×906 / 8.5MB / H.264+AAC）
  - 再エンコード目標: H.264 2〜3MB + 可能なら WebM(VP9/AV1) 併用、SP向け720p版
  - 音声トラックは削除（muted必須のため不要）

## 6. コンテンツ管理（データ駆動）

非エンジニアの実行委員が更新できることを優先する。

| データ | 形式（案） | 更新頻度 |
|---|---|---|
| news | Markdown（1件1ファイル）or JSON | 開催前〜当日 高 |
| booths.json | JSON（名称・団体・説明・タグ・場所） | 確定後ほぼ固定 |
| timetable.json | JSON（日・時刻・場所・企画名） | 確定後ほぼ固定 |
| sponsors.json | JSON（名称・ロゴ・URL・ランク） | 随時追加 |

- 更新フロー: GitHub上で編集 → push → Cloudflare Pages 自動ビルド・デプロイ
- スキーマは実データのサンプルをもらってから確定する

## 7. 技術・非機能要件

- **Astro**（静的生成）+ 素のCSS を土台に、インタラクション/アニメーションが必要な箇所のみ React + Framer Motion（`motion`）を「アイランド」として追加（例: ハンバーガーメニュー開閉、セクションのスクロール登場演出）。それ以外のJSは booth 絞り込み・OP制御のみ
- **Cloudflare Pages** にデプロイ（GitHub連携の自動ビルド）
- パフォーマンス: 4G回線で LCP 2.5秒以内を目安。画像は圧縮 + `loading="lazy"`、OPムービーはスプラッシュ以外でプリロードしない
- OGP画像・favicon（ロゴ流用）・タイトル/description 整備
- アクセシビリティ: 文字コントラスト確保、画像alt、reduced-motion 対応（マーキー・OP停止）
